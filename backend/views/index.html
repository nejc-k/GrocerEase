API Endpoints

Here are the available endpoints for the API:

Articles Endpoints

  • GET /api/article

    Get all articles.

  • GET /api/article/:id

    Get a single article by ID.

  • POST /api/article

    Create a new article.

  • PUT /api/article/:id

    Update an article by ID.

  • DELETE /api/article/:id

    Delete an article by ID.

  • GET /api/article/store/:id

    Get all articles from the specified store.

  • GET /api/article/category/:category

    Get all articles from the specified category.

  • POST /api/article/query

    Query articles by parameters given in the request body.

    Example: 
                        {
                        "category": "breadAndPastry",
                        "store": "Mercator",
                        "min_price": 0.50,
                        "max_price": 2.49,
                        "title": "Žemlja"
                    }

    Given example would query and return all articles form "Mercator" store under the "breadAndPastry" category with price in range "0.50" and "2.49" and where title contains substring "Žemlje", case-insensitive.

Stores Endpoints

  • GET /api/stores

    Get all stores.

  • GET /api/stores/:id

    Get a single store by ID.

  • POST /api/stores

    Create a new store.

  • PUT /api/stores/:id

    Update a store by ID.

  • DELETE /api/stores/:id

    Delete a store by ID.

Users Endpoints

  • POST /api/users/register

    Register a new user.

  • POST /api/users/login

    Login a user.

  • POST /api/users/logout

    Logout a user.

  • GET /api/users/profile

    Get user profile.
